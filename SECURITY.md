# Security Policy

## Supported Versions

| Version | Supported |
|---|---|
| Latest release | ✅ |
| Older releases | ❌ |

## Reporting a Vulnerability

If you discover a security vulnerability, please report it privately via
**GitHub's Private Vulnerability Reporting**:

https://github.com/bhave-sh/aidem/security/advisories/new

**Please do NOT open a public GitHub issue.**

Include the following details:

- Type of vulnerability
- Affected file(s) or component(s)
- Steps to reproduce
- Proof-of-concept (if available)
- Potential impact

### What to expect

1. **Acknowledgment** within 48 hours
2. Assessment and determination of severity
3. Regular updates on progress
4. Credit in the advisory (with your permission)

### Scope

The following are in scope:
- The `aidem` CLI tool and its source code
- Installation and setup scripts

The following are out of scope:
- Third-party tools added via `aidem registry add` (report to the upstream project)
- Dependencies (report to the upstream project)
